# Changelog

## 0.1.4 - 2026-03-31

- split the textarea editor into deeper button-style `Field` and `Writing` sections
- added quick writing-mode variants for reply, message, editorial, and code-like surfaces
- wired description and success messaging through live preview and exported React while preserving char-count and floating-label parity

## 0.1.3 - 2026-03-31

- expanded the textarea state model and editor surface with additional production attributes such as `id`, `title`, `tabIndex`, `dir`, `lang`, `inputMode`, `enterKeyHint`, `autoCapitalize`, `autoCorrect`, `minRows`, and `maxRows`
- kept live preview and exported React output aligned with the broader textarea attribute surface

## 0.1.2 - 2026-03-31

- expanded the preset system from starter packs into larger generated button-style preset catalogs
- aligned the preset browser and preview reset flow more closely with the button-component product model
- preserved clean lint, typecheck, and production build verification after the canon sync

## 0.1.1 - 2026-03-31

- added a searchable preset catalog with full-state preset apply flow
- aligned the textarea studio more closely with the button-style UI Foundry editing model
- preserved clean lint, typecheck, and production build verification after the preset rollout

## 0.1.0 - 2026-03-30

- stabilized the standalone textarea app so build, lint, and typecheck pass
- synced shared layout and control primitives to the UI Foundry standard
- preserved a self-contained textarea playground and export workflow
